# FUTURE_CS_03
# API Security Risk Analysis

## Project Overview
This project is a read-only API Security Risk Analysis performed on a public demo API. The goal is to identify common API security risks and explain them in simple business language.

## API Tested
JSONPlaceholder Demo API  
https://jsonplaceholder.typicode.com

## Tools Used
- Postman
- Browser DevTools
- MS Word / Google Docs
- GitHub

## Scope
Only public demo API endpoints were tested.  
No exploitation, bypassing, flooding, or harmful testing was performed.

## Methodology
1. Selected a public demo API
2. Reviewed available endpoints
3. Tested API requests using Postman
4. Inspected headers, status codes, and response data
5. Identified possible API security risks
6. Classified risks based on severity
7. Suggested remediation steps

## Key Findings
- Open unauthenticated endpoints
- Excessive data exposure
- Missing rate limit information
- Weak object-level access control simulation
- Input validation risk
- Missing security headers

## Deliverables
- API Security Risk Analysis Report
- Postman screenshots
- README documentation

## Ethical Note
This project was performed only on public demo APIs for learning and reporting purposes.
